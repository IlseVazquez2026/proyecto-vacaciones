Diseña la interfaz con un estilo tipo dashboard moderno, limpio y profesional, similar a aplicaciones administrativas actuales. La interfaz debe ser clara, visualmente ordenada y fácil de interpretar.

📌 ESTILO GENERAL
Diseño tipo dashboard administrativo moderno.
Uso de tarjetas (cards) para agrupar información.
Espacios en blanco bien distribuidos.
Evitar saturación visual.
Priorizar legibilidad sobre decoración.
Interfaz pensada para uso diario (no solo visualmente atractiva).
🎨 PALETA DE COLORES

Usar una paleta similar a la del ejemplo:

Colores base
Fondo general: blanco o gris muy claro (#F5F6FA o similar)
Tarjetas: blanco con sombras suaves
Bordes: gris claro (#E0E0E0)
Colores principales
Primario: morado o azul moderno (#6C5CE7 o similar)
Secundario: rosa/magenta suave (#E84393 o similar)
Complementario: azul claro (#00B894 o #0984E3)
Colores de estado
Éxito: verde (#00B894)
Advertencia: naranja (#FDCB6E)
Error: rojo (#D63031)
Información: azul (#0984E3)

👉 Importante:

No usar colores muy saturados.
Mantener consistencia en toda la app.
Usar colores para indicar estado, no solo decoración.
🔤 TIPOGRAFÍA
Fuente recomendada: sans-serif moderna (ej. Inter, Roboto, Open Sans).
Tamaños jerárquicos claros:
Títulos principales: 18–22 px
Subtítulos: 14–16 px
Texto general: 12–14 px
Etiquetas: 11–12 px
Usar negritas solo para información importante.
Evitar texto largo en una sola línea.

📊 VISUALIZACIÓN DE DATOS

La app debe incluir elementos visuales tipo dashboard:

Gráficas simples (línea o barras)
Indicadores numéricos (KPIs)
Tarjetas resumen

Ejemplo aplicado a vacaciones:

total de días disponibles;
días usados;
días pendientes;
personas de vacaciones en el mes.
📅 CALENDARIO

El calendario debe:

ser visualmente limpio;
mostrar claramente nombres por día;
permitir identificar rápidamente ocupación;
usar colores suaves para diferenciar estados;
evitar saturación de texto;
permitir cambiar de mes fácilmente.

Menú lateral izquierdo (como en la imagen)
Íconos simples por sección
Secciones claras:
Dashboard
Colaboradores
Vacaciones
Calendario

La navegación debe ser intuitiva y constante.
🧩 JERARQUÍA VISUAL
Lo más importante debe verse primero.
Agrupar información relacionada.
Separar visualmente secciones.
No mezclar demasiada información en una sola vista.
⚙️ COMPORTAMIENTO DE LA INTERFAZ
Respuestas rápidas al usuario.
Cambios visibles al editar datos.
Mensajes claros (éxito, error, advertencia).
Confirmación en acciones importantes.

profesional;
moderna;

fácil de usar;
similar a dashboards administrativos actuales.

un Excel trasladado a pantalla;

rioriza claridad visual, orden y experiencia de usuario por encima de complejidad gráfica.

Diseña la aplicación con un layout tipo dashboard moderno, claro, ordenado y fácil de usar. La estructura debe facilitar la navegación, la consulta rápida de información y el uso diario sin complejidad.

📌 ESTRUCTURA GENERAL

La aplicación debe tener una estructura fija:

Menú lateral izquierdo (navegación principal)
Área principal de contenido a la derecha
Encabezado superior opcional para acciones generales

El layout debe mantenerse consistente en todas las pantallas.

📌 MENÚ LATERAL

Debe ubicarse del lado izquierdo y mantenerse visible en toda la aplicación.

Debe incluir:

iconos simples;
nombres claros;
secciones bien definidas.

Secciones sugeridas:

Dashboard
Colaboradores
Vacaciones
Calendario
Histórico
Configuración

El menú debe ser limpio, compacto y fácil de navegar.

📌 ÁREA PRINCIPAL

Debe ser el espacio donde se muestran todas las vistas y módulos.

Debe utilizar:

tarjetas (cards);
bloques de información separados;
espacios en blanco adecuados;
alineación consistente.
📌 ESTRUCTURA DEL DASHBOARD

El dashboard principal debe organizarse en tres niveles:

1. Parte superior (indicadores)

Mostrar tarjetas con información clave:

días disponibles;
días usados;
saldo;
número de colaboradores en vacaciones.

Estas tarjetas deben ser visibles de inmediato.

2. Parte central (visualización principal)

Debe incluir:

calendario mensual (elemento principal);
o gráfica resumen si se requiere.

El calendario debe ocupar el espacio principal.

3. Parte inferior (detalle)

Mostrar información complementaria:

lista de próximas vacaciones;
actividad reciente;
resumen por colaborador.
📌 LAYOUT DE PANTALLA DE COLABORADORES

Debe incluir:

Parte superior:
buscador;
filtros (activo/inactivo, área, etc.);
botón de alta de colaborador.
Parte central:
tabla de colaboradores con columnas como:
nombre;
fecha de contratación;
estatus;
saldo de vacaciones.
Parte lateral o secundaria (opcional):
detalle del colaborador seleccionado.
📌 LAYOUT DE REGISTRO DE VACACIONES

Debe ser claro y rápido de usar.

Parte superior:
selección de colaborador;
selección de periodo.
Parte central:
opción de seleccionar:
rango de fechas;
o múltiples fechas específicas.
Parte inferior:
resumen automático de días seleccionados;
botón de guardar.
📌 LAYOUT DE HISTÓRICO INDIVIDUAL

Debe mostrar:

Parte superior:
datos del colaborador;
saldo actual.
Parte central:
tabla con:
fechas oficiales;
fechas reales;
estatus;
días usados.
Parte inferior (opcional):
resumen por año o periodo.
📌 LAYOUT DEL PANEL GENERAL

Debe ser una vista tipo tabla o dashboard.

Parte superior:
filtros:
año;
colaborador;
área.
Parte central:
tabla con:
colaborador;
días asignados;
días usados;
saldo;
estatus.

Debe permitir identificar rápidamente el estado de todo el personal.

📌 LAYOUT DEL CALENDARIO

Debe ser visualmente claro y fácil de leer.

Debe mostrar:

días del mes;
nombres de colaboradores en cada día;
colores suaves para diferenciar estados.

Debe permitir:

cambiar de mes;
identificar rápidamente ocupación diaria.

El calendario debe ser el elemento visual más importante.

📌 ORGANIZACIÓN VISUAL
Usar tarjetas para agrupar información.
Mantener márgenes y espacios consistentes.
No saturar las pantallas.
Mostrar solo la información necesaria en cada vista.
Separar claramente secciones.
📌 JERARQUÍA
Lo más importante debe verse primero.
Indicadores arriba.
Visualización principal en el centro.
Detalles abajo.
📌 CONSISTENCIA
Todas las pantallas deben seguir la misma estructura.
Botones en posiciones similares.
Filtros en la parte superior.
Tablas y formularios con diseño uniforme.
📌 OBJETIVO DEL LAYOUT

El layout debe permitir:

navegación intuitiva;
lectura rápida de información;
captura eficiente;
consulta clara;
apariencia profesional.

Debe evitar:

pantallas desordenadas;
exceso de información;
estructuras tipo Excel;
navegación confusa.
